# Идеи по функционалу

Уже есть и не трогаем: избранные курорты, отзывы/рейтинги на курорты, сравнение, карта.

## Сильные кандидаты

1. **Снежные алерты** — подписка на курорт → email, когда прогноз обещает снегопад > N см.
   Данные уже есть (snowfall_sum в weather-service), избранное как база подписки уже есть.
   Нужно: cron-проверка + SMTP (тот же SMTP закроет password reset из PRODUCTION_TODO).
2. **Калькулятор поездки** — «7 дней на двоих»: скипассы + отель + уроки = смета.
   Все цены уже в сервисах, паттерн — страница Compare. Чисто фронтовая фича.
3. **Отзывы на отели** — на курорты есть, на отели нет. Тот же паттерн, что в resort-service.

## Средний приоритет

4. **Запись на уроки** — сейчас уроки только каталог. Кнопка «записаться» + список записей
   у пользователя и инструктора.
5. **Статусы объявлений на барахолке** — «продано»/«активно» + контакт продавца,
   иначе старые объявления висят вечно.
6. **Глобальный поиск** — одна строка в шапке по курортам/отелям/снаряге.

## Потом

7. **Дневник катаний** — «отметить день на склоне»: даты, курорт, спуски.
   Естественный источник данных для stats-service — проектировать вместе с ним.

Рекомендуемый порядок: 1 → 2 → 3.
